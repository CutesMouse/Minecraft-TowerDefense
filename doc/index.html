前言

關於本專案

專案簡介

本專案旨在藉由實作電腦遊戲Minecraft的擴充插件，創造一個獨立的塔防遊戲。

組員

程式組組員共兩位：吳昱辰與徐粲桓。工作分配方面為吳昱辰負責主要插件開發，徐粲桓負責遊戲內容設計。

遊戲機制

藉由擴充插件建立遊戲基本架構，監控並管理玩家的行為，並針對玩家不同的選擇做出相對應的回應。將遊戲內容分為數個部分進行管理：

  • 地圖結構：存取地圖上的結構，並管理遊戲過程中的地圖結構變動
  • 防禦塔：管理防禦塔的攻擊機制、建造與拆除、攻擊粒子特效等細項
  • 怪物：管理遊戲中怪物的行為，如移動與轉向、類別數值等。
  • 其它：管理遊戲中的其他細項，如記分板、進度條等。

藉由不同部分間的相互控制與運作，達成進行遊戲的效果。

遊戲規則

遊戲可單人或多人進行，玩家藉由在地圖中建造防禦塔抵禦怪物的進攻，而怪物以波數的模式生成。怪物若在死亡前抵達路徑終點，將耗損玩家血量一點，遊戲持續進行到玩家血量耗盡。

怪物簡介

遊戲中怪物共有19種，但大多數都是Minecraft原版設計的沿用，礙於篇幅在此僅介紹三種具特殊設定的怪物。其它怪物的血量、速度屬性會在 #遊戲平衡調整 以表格呈現。

  • Slime

Slime共有三種大小等級，在死亡之後會隨機分裂成0~2隻低一級的Slime。分裂後的Slime會有較少的血量，但速度會提升。

  • Golem

Golem是每5波生成一次的小Boss。

  • Wither

Wither是每10波生成一次的大Boss，可回復玩家血量至多5點（不超過最高血量）

防禦塔簡介

遊戲中防禦塔一共有10種，分為三類：

  • 攻擊類

這類防禦塔的特點在於攻擊力高，攻擊延遲中等，這類防禦塔有些只能進行單體攻擊，如巨錘瑞斯塔與火焰塔；有些則具有範圍攻擊的效果，如藥水塔與炸藥塔，另外藥水塔還會附加緩速與中毒效果。

  • 範圍類

此類防禦塔攻擊範圍較大，可以攻擊到距離較遠的敵人，傷害最高，但攻擊延遲也最高。如弓箭塔、長弓塔與狙擊塔。

  • 攻速類

這類防禦塔的攻擊速度很快，可以在短時間內攻擊同一敵人多次，能對一整群的怪物造成相對平均的傷害，且雪球塔還具有緩速效果，而缺點便是單次攻擊傷害較低。如十字弓塔、雪球塔與機關槍塔。

插件實作

地圖結構

總覽

地圖結構我們的實作方式是先建立 Map 介面，然後以 CraftMap 抽象類別作為 Map 介面的實作，並以該類別作為遊戲中兩張地圖的母模板。

  • CraftMap 建構子與全域變數

為了節省效能，建構子中就已直接把所有地圖結構之必要物件準備好。為此需要在類別中新增一些抽象函式，像是 initTowers()、initCheckpoints() 等。在建構子中直接執行，如此一來需要時就可以直接回傳準備好的資料。



















    // @全域變數: 這些變數會在 initTowers(), initCheckpoints() 等函式初始化
    protected ArrayList<TowerSlot> towerSlots;
    protected ArrayList<Mob> moblist;
    protected ArrayList<Tower> towers;
    protected ArrayList<CheckPoint> cps;
    protected LinkedList<Location> castleBlocks;
    protected ArrayList<Location> finalCastleBlocks;

    public CraftMap() {
        // 初始化所有全域變數
        initTowers();
        initCheckpoints();
        initCastleBlocks();
        initFinalCastleBlocks();

        // 以內積獲得Checkpoints間的角度
        castCheckpointAngles();
    }

怪物生成位置

  • 一般怪物的生成

透過怪物的類別，我們以 Class.newInstance() 的方式產生新的 Mob 物件，並執行 Mob.spawn() ，使怪物順利生成於地圖中的指定位置。


















    @Override
    public Mob spawnMob(Class<? extends Mob> mob) {
        try {
            // 創建一個怪物物件
            Mob mobEntity = mob.newInstance();
            // 呼叫怪物類別的spawn方法，將怪物生成於地圖的怪物生成位置
            mobEntity.spawn(getMobSpawnLocation());
            // 將怪物加入場上怪物清單
            moblist.add(mobEntity);
            return mobEntity;
        } catch (InstantiationException | IllegalAccessException e) {
            e.printStackTrace();
        }
        return null;
    }
  • 分裂後Slime生成

由於分裂後的Slime召喚方式與一般的怪物不同，我們另設一個 spawnSpiltSlime() 函式來處理。

























    @Override
    public void spawnSpiltSlime(Location parentLocation, int afterSize, int amount) {
        if (amount == 0) return; // 不分裂
        // 獲取該波波數資訊
        Wave wave = TowerDefense.getInstance().getCurrentWave();
        if (wave == null) return; // 無波數資訊
        // 獲取該波怪物加強參數
        int strengthen = wave.getStrengthen();
        // 加入位置隨機變數，讓Slime分裂後的位置不重疊
        Random r = new Random(); 
        // 生成指定數量之Slime
        for (int i = 0; i < amount; i++) { 
            Mob mob = CraftMob.SlimeSpilt(afterSize);
            // 獲取分裂後Slime怪物實例
            if (mob == null) return; // 已經分裂成最小
            mob.strengthen(strengthen); // 強化分裂後Slime
            // 生成小Slime實例於大Slime死亡位置
            mob.spawn(randomOffset(parentLocation,r,0.1));
            // 將小Slime面向設成大Slime死亡時面向
            mob.setFacingDegree(parentLocation.getYaw());
            // 將分裂後Slime加入場上怪物清單
            moblist.add(mob);
        }
    }

轉彎節點

轉彎節點的類別取名叫Checkpoint，儲存在Map中。每個節點包含兩個訊息，分別是節點座標與轉向角度。當偵測到怪物接近，就設定怪物的面向角度，令怪物轉向。































public class CheckPoint {
    private Location loc; // 節點座標
    private double yaw; // 轉向角度
    
    public CheckPoint(double x, double y, double z) {
        this.loc = new Location(null,x,y,z);
    }
    
    /*
     * @呼叫: CraftMap 建構子
     * @簡介: 連接各轉彎節點
     * 
     * @point: 目標節點
     * */
    public void connectTo(CheckPoint point) {
        yaw = MathTool.locationToDegree(loc,point.loc);
    }

    /*
     * @呼叫: 判定是否經過節點
     * 
     * @def: 距離最小值
     * */

    public boolean isPassBy(Location loc,double def) {
        this.loc.setWorld(loc.getWorld());
        return distance(loc,this.loc) < def;
    }
}

防禦塔建造格

  • 建造格管理

防禦塔建造格由TowerSlots管理。這裡存放著每個儲存格的座標、大小、與面向，這些數據可在蓋塔時用來呼叫建築相關的函式。














public class TowerSlot {

    private Location loc;
    
    private TowerType type; // 大小，區分為5x5或3x3
    private Direction outDirection; // 面相，區分為東、南、西、北
    private Tower tower;

    public TowerSlot(Location loc, TowerType type, Direction outDirection) {
        this.loc = loc;
        this.type = type;
        this.outDirection = outDirection;
    }
  • 蓋塔與建塔

玩家在蓋塔時，會從TowerSlot.buildTower()執行，執行後會呼叫Tower.build()來完成建造動作。接著還需要在Map中註冊防禦塔，才能有效管理與更新。














    
    public void buildTower(Tower tower) {
        tower.build(this); // 將防禦塔本體建構出來
        Main.map.getTowers().add(tower); // 註冊新塔
        this.tower = tower;
    }

    public void removeTower() {
        if (tower == null) return;
        tower.remove();
        Main.map.getTowers().remove(tower); // 取消註冊
        tower = null;
    }
  • 獲取邊框

在TowerSlot函式中也配有getOutline()這樣的函式，可以直接獲取外框座標清單。我們將以此來顯示玩家指向的TowerSlot。


















    public ArrayList<Location> getOutline() {
        int blockDiff = type.equals(TowerType.THREE_BY_THREE) ? 1 : 2;
        ArrayList<Location> locs = new ArrayList<>();
        int x = loc.getBlockX();
        int y = loc.getBlockY();
        int z = loc.getBlockZ();

        for (int dx = -blockDiff; dx <= blockDiff; dx++) {
            for (int dz = -blockDiff; dz <= blockDiff; dz++) {
                if (dx == -blockDiff || dz == -blockDiff || 
                    dx == blockDiff || dz == blockDiff)
                locs.add(new Location(loc.getWorld(), x + dx, y, z + dz));
            }
        }
        return locs;
    }

怪物

怪物的實作架構與防禦塔大致相同，我們先建立了 Mob 介面，規範了怪物的要素。然後再以 CraftMob 實作該介面。

對怪物的實時控制放在CraftMap中，每tick（0.05秒、50毫秒）執行一次，會一次控制怪物移動、上方血條與更新效果。












































    /**
     * @呼叫: 遊戲開始時
     * */
    
    @Override
    public void start() {
        new BukkitRunnable() {
            @Override
            public void run() {
                // @如果: 遊戲尚未開始，則結束執行緒
                if (Main.map == null || !Main.map.equals(CraftMap.this)) {
                    this.cancel();
                    return;
                }
                tickEvent();
            }
        }.runTaskTimer(Main.getPlugin(Main.class),0L,1L);
    }
    
    /** 
     * @呼叫: 每tick (0.05秒、50毫秒) 呼叫一次，由 start() 執行
     * */
    @Override
    public void tickEvent() {
        if (moblist.size() > 0) { // @如果: 場上還有怪物
            ArrayList<Mob> copy = new ArrayList<>(moblist); 
            // 複製一份怪物清單，避免怪物死亡、自動從清單移除時，產生錯誤
            
            for (Mob mob : copy) {
                if (mob == null) continue;
                
                // 移動怪物。
                mob.onEntityMove();
                
                // 更新血條。
                mob.updateDisplayName();
                
                // 更新效果。
                mob.effectUpdate();
                
            }
        }
    }

移動路徑控制

怪物的移動可以視為一個變動的向量，具有大小與方向。
在方向方面，考量到兩個 Checkpoint 之間的區段使用的面向角度是固定的，而到 Checkpoint 時意味著進入路徑的下一區段，需要轉換面向角度。我們設計的方式是取得兩紀錄點的方向向量並與z軸單位向量以內積取夾角，並以該夾角改變怪物面向角度。而以上過程均在方法 locationToDegree 當中完成。
